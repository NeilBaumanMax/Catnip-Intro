# LAYER_CONTRACT

## 当前阶段

当前只完成 Phase 0：工程脚手架。

本文件定义各层允许和禁止的行为，后续开发必须遵守。

## frontend

允许：

- 实现前台页面。
- 实现后台管理页面。
- 调用 backend API。
- 展示 backend 返回的数据。
- 使用 backend 返回的图片 URL 展示图片。
- 做基础表单校验和用户交互。

禁止：

- 直接连接 SQLite。
- 直接读写 `data/`。
- 直接读写 `uploads/` 文件系统。
- 在前端保存明文管理员密码。
- 将后台鉴权只放在前端完成。
- 私自定义不经过 backend 的数据来源。

## backend

允许：

- 提供登录接口。
- 提供产品、案例、留言、网站设置接口。
- 提供图片上传接口。
- 提供 uploads 静态文件访问。
- 读写 SQLite。
- 校验请求参数。
- 控制后台接口鉴权。
- 统一处理错误响应。

禁止：

- 在 SQLite 中保存真实图片二进制文件。
- 跳过输入校验直接写入数据库。
- 让未鉴权请求访问后台管理接口。
- 将业务数据写入 frontend 目录。
- 将上传图片写入 frontend 目录。

## SQLite

允许：

- 保存 Admin 数据。
- 保存 Product 数据。
- 保存 Case 数据。
- 保存 Message 数据。
- 保存 SiteSetting 数据。
- 保存图片路径、排序、状态、时间字段等结构化数据。

禁止：

- 保存真实上传图片文件。
- 作为浏览器可直接访问的接口。
- 存放临时构建产物。
- 存放前端静态资源。

## uploads

允许：

- 保存后台上传的真实图片文件。
- 按后端规则组织图片路径。
- 由 backend 暴露静态访问路径。

禁止：

- 保存 SQLite 数据库文件。
- 保存业务配置文档。
- 保存源码。
- 由 frontend 直接以文件系统路径读写。

## integration

允许：

- 定义 frontend 与 backend 的 API 契约。
- 定义图片 URL 访问规则。
- 定义错误码和响应结构。
- 定义登录态传递方式。

禁止：

- 绕开 backend 直接集成 frontend 与 SQLite。
- 绕开 backend 直接集成 frontend 与 uploads 文件系统。
- 在未确认技术栈前锁定具体实现细节。

# CONSTRUCTION_PLAN

## 当前阶段

当前只完成 Phase 0：工程脚手架。

## Phase 0：工程脚手架

目标：

- 创建项目基础目录。
- 创建工程文档。
- 明确项目目标、功能边界、层级职责和验收标准。

允许：

- 创建 Markdown 文档。
- 创建空工程目录和 README。

禁止：

- 写业务代码。
- 安装依赖。
- 创建新 worktree。
- 切换 Git 分支。

完成标准：

- 所有指定目录存在。
- 所有指定文档存在。
- 文档明确 Phase 0 范围。

开工：

- 检查 Git 状态。
- 备份当前仓库状态。
- 读取 Phase 0 相关文档。
- 写 Phase 0 开工计划。

施工：

- 只创建目录和工程文档。
- 不写业务代码。

测试：

- 检查指定目录和文档是否存在。
- 检查没有依赖、业务代码、新 worktree 或分支切换。

失败处理：

- 记录失败项和原因。
- 回到 Phase 0 文档或目录施工。
- 修复后重新测试。

收尾：

- 写日志、测试记录、进度和交接。
- 修正文档漂移。
- 提出 Phase 1 施工建议。
- 提交或上传 GitHub，并记录结果。
